# Hybrid-Enhanced Traffic Signal Control

This repository contains the final implementation of the **Hybrid-Enhanced** adaptive traffic signal control method for an isolated four-way intersection.

The final method is:

- **Hybrid-Enhanced Pri-Hyper DDQN**

It combines:

- Pri-DDQN style prioritized double dueling Q-learning
- a temporal hypergraph encoder over traffic movements
- a pressure-aware action prior
- conservative phase-switch filtering

## 7. How the Final Hybrid-Enhanced Code Works

The Hybrid-Enhanced method runs on the 4-action movement benchmark in:

- [main_experiment_6action/environments/traffic_env.py](/C:/Users/Ashritha/Downloads/final_rl/traffic_dqn_fixed/traffic_dqn/main_experiment_6action/environments/traffic_env.py)

### Main pipeline

1. Read the raw traffic state from the simulator.
2. Build a temporal state window of recent steps.
3. Convert movement features into a hypergraph-style node representation.
4. Encode movement relations and short-term history.
5. Estimate Q-values with a double dueling Q-network.
6. Compute a pressure-aware prior over signal actions.
7. Blend learned Q-values with the prior.
8. Apply conservative switching logic.
9. Execute the final action in the environment.

### Main code locations

- [hybrid_enhanced_benchmark.py](/C:/Users/Ashritha/Downloads/final_rl/traffic_dqn_fixed/traffic_dqn/hybrid_enhanced_benchmark.py)

### Issue: `torch` import problems on Windows

Use the CPU build:


### Issue: CUDA warnings

This only affects paper compilation, not model training. Install a LaTeX distribution such as:

- TeX Live
- MiKTeX

The final script trains before evaluating. That is expected. Outputs appear after completion in:

- `outputs/hybrid_enhanced/`
